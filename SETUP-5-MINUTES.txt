FIX LOGIN / DASHBOARD IN A FEW MINUTES (you, not the client)

The public site works without DB. Sign-in needs PostgreSQL + env vars.

STEP 1 — Database
  https://neon.tech → New project → copy connection string → DATABASE_URL

STEP 2 — Host env vars
  Add in your hosting dashboard (Railway, Render, Fly, VPS shell, Netlify — same vars):

    DATABASE_URL   = from Neon
    AUTH_SECRET    = long random string
    ADMIN_EMAIL    = author@example.com
    ADMIN_PASSWORD = your-password

Optional URL (many hosts set automatically):
    AUTH_URL = https://your-domain.TLD/
    NEXT_PUBLIC_SITE_URL = same as AUTH_URL

STEP 3 — First-time DB (pick one)

  A) SSH / one-off on the host before start:
       npx prisma migrate deploy && npm run db:seed

  B) Auto on boot — set RUN_DB_SETUP_ON_BOOT=1 (see README).

STEP 4 — Deploy / restart
STEP 5 — Sign in at /login
